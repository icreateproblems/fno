# Contributing to FastNewsOrg\n\nThank you for considering contributing to FastNewsOrg. This document provides guidelines and instructions for contributing.\n\n## Code of Conduct\n\nBe respectful, constructive, and professional in all interactions.\n\n## How to Contribute\n\n### Reporting Bugs\n\n1. Check existing issues first\n2. Use issue template\n3. Provide:\n   - Clear description\n   - Steps to reproduce\n   - Expected vs actual behavior\n   - Environment details\n   - Log output\n\n### Suggesting Features\n\n1. Search existing feature requests\n2. Explain use case\n3. Describe proposed solution\n4. Consider alternatives\n\n### Pull Requests\n\n1. Fork repository\n2. Create feature branch: `git checkout -b feature/your-feature-name`\n3. Make changes\n4. Test thoroughly\n5. Commit with clear messages\n6. Push to your fork\n7. Open pull request\n\n## Development Setup\n\n```bash\ngit clone https://github.com/sakshyambanjade/fastnewsorg.git\ncd fastnewsorg\npython -m venv venv\nsource venv/bin/activate  # or .\\venv\\Scripts\\Activate.ps1 on Windows\npip install -r requirements.txt\ncp .env.example .env\n# Edit .env with your credentials\n```\n\n## Code Style\n\n- Follow PEP 8\n- Use meaningful variable names\n- Add docstrings to functions\n- Comment complex logic\n- Keep functions focused and small\n\n## Testing\n\nBefore submitting PR:\n\n```bash\n# Test all scripts\npython scripts/test_setup.py\npython scripts/fetch_news.py\npython scripts/post_instagram.py\n\n# Check syntax\npython -m py_compile scripts/*.py *.py\n\n# Verify no sensitive data in commits\ngit diff\n```\n\n## Commit Guidelines\n\n**Format:**\n```\ntype: brief description\n\nDetailed explanation (optional)\n\nCloses #issue-number (if applicable)\n```\n\n**Types:**\n- feat: New feature\n- fix: Bug fix\n- docs: Documentation\n- style: Formatting\n- refactor: Code restructuring\n- test: Adding tests\n- chore: Maintenance\n\n**Examples:**\n```\nfeat: add Bloomberg RSS source\n\nAdded Bloomberg news feed to FREE_RSS_SOURCES\nwith financial category and 15s timeout.\n\nCloses #42\n```\n\n## Areas for Contribution\n\n### High Priority\n\n- Additional news sources\n- Improved error handling\n- Better AI prompt engineering\n- Performance optimizations\n- Documentation improvements\n\n### Medium Priority\n\n- Unit tests\n- Integration tests\n- Additional CI/CD platforms\n- Dashboard for monitoring\n- Webhook notifications\n\n### Low Priority\n\n- UI improvements\n- Additional social platforms\n- Analytics features\n- Mobile app\n\n## Adding News Sources\n\nEdit `config.py`:\n\n```python\nFREE_RSS_SOURCES = {\n    \"Source Name\": {\n        \"url\": \"https://example.com/rss\",\n        \"category\": \"general\",  # or financial, tech, regional\n        \"region\": \"global\",     # or specific region\n        \"timeout\": 10\n    }\n}\n```\n\nTest:\n```bash\npython scripts/fetch_news.py\n```\n\n## Improving AI Captions\n\nEdit `scripts/groq_caption.py`:\n\n```python\nsystem_prompt = \"\"\"\nYour improved prompt here\n\"\"\"\n```\n\nTest:\n```bash\npython -c \"from scripts.groq_caption import generate_caption; print(generate_caption('Test headline'))\"\n```\n\n## Documentation\n\nDocumentation improvements always welcome:\n\n- Fix typos\n- Clarify confusing sections\n- Add examples\n- Update outdated info\n- Translate to other languages\n\n## Questions\n\nNot sure about something? Open a discussion or issue.\n\n## License\n\nBy contributing, you agree your contributions will be licensed under the MIT License.\n
